# Лабораторная работа 7: Архитектура, слои и DDD-lite

Реализация системы оплаты заказа с использованием слоистой архитектуры и принципов DDD-lite.

## Описание проекта

Проект демонстрирует реализацию системы оплаты заказа с четким разделением на доменный, прикладной и инфраструктурный слои.

## Функциональность

- Создание заказов с товарными позициями
- Оплата заказов с проверкой бизнес-правил
- Управление статусами заказов
- Валидация доменных инвариантов

## Основные особенности

### Доменные инварианты:
1. Нельзя оплатить пустой заказ
2. Нельзя оплатить заказ повторно
3. После оплаты нельзя менять строки заказа
4. Итоговая сумма равна сумме строк заказа

### Слои архитектуры:
- **Domain**: бизнес-логика и доменная модель
- **Application**: use-case оплаты заказа
- **Infrastructure**: реализации репозитория и платежного шлюза
- **Tests**: модульные тесты для use-case
